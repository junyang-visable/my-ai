# 样式与响应式调整 Prompt 模板

## 使用场景

在**不改动 HTML/Vue 模板结构**（或仅允许极小 class 绑定调整且需说明）的前提下，修改组件或页面的 **CSS / 响应式布局**。颜色须走设计令牌（CSS 变量），断点与团队约定一致。

---

## Prompt 模板

```markdown
## 上下文

- 样式工具栈：`[替换：如 SCSS、PostCSS、Tailwind、UnoCSS 等]`；全局颜色与主题变量定义在 `assets/styles/variables.css`（或项目实际变量文件：`[替换：路径]`）。
- 目标组件 / 页面文件：`[替换：.vue 路径]`
- 当前设计稿或截图说明（可选）：`[替换：Figma 链接或简要描述]`

## 任务

调整 **`[替换：组件或区块名称]`** 的样式，具体需求：

- `[替换：例如：标题与副标题间距从 8px 改为 16px]`
- `[替换：例如：移动端卡片改为单列，平板两列，桌面三列]`
- `[替换：例如：hover 态边框使用主题色强调色]`

## 约束

1. **禁止硬编码颜色值**（无特殊情况不要使用 `#rrggbb`、`rgb()` 写死品牌色）；统一使用 `var(--xxx)` 引用 `variables.css` 中已有变量；若确需新令牌，在输出中**列出建议在 variables.css 新增的变量名与语义**。
2. **不要改变 HTML 结构**：不增删 DOM 节点、不改标签层级；若必须增加 wrapper 类名，请说明理由并视为例外。
3. **响应式断点约定**：
   - Mobile：`< 768px`
   - Tablet：`768px - 1024px`（含边界与项目现有 media query 习惯保持一致时，在输出中用注释说明）
   - Desktop：`> 1024px`
4. 不引入不存在的样式依赖；优先使用现有 BEM / 模块前缀等命名约定。
5. 动画与过渡若涉及 `prefers-reduced-motion`，按项目惯例处理（无惯例则保持简单 transition）。

## 输出要求

1. **仅输出样式相关的 diff 形式变更**（可用 unified diff 或按「文件路径 + 删除块 / 新增块」清晰标注），不要重复粘贴未修改的整文件。
2. 若涉及 `variables.css` 的建议增量，单独给出该文件的 diff 片段。
3. 用一两句话说明 **如何在三种断点下目检验收**。
```

---

## 使用说明

- **先确认变量名**：打开 `variables.css` 把实际 token 名贴进「上下文」，模型更少臆造 `--primary-500` 这类不存在的名字。
- **结构不能动时**：若视觉需求必须改结构，应另开任务走产品/设计确认，本模板默认严格限制 DOM。
- **Tailwind 项目**：在「上下文」写明是否使用任意值、是否与 design token 映射在 config 中，避免生成被 lint 禁止的类名。
- **打印与无障碍**：若调整涉及对比度，可要求输出中注明对比度是否仍满足 WCAG AA（粗略即可）。
